# Contributing to The Technical Writer's Edge

Thank you for helping improve this guide. It exists because technical writers share openly — contributions of all sizes are welcome.

---

## Ways to Contribute

### Suggest a tip or correction
Open an issue using the [Suggest a Tip](../../issues/new?template=suggest_tip.md) template. Describe the tip, why it's useful, and if it came from personal experience or a community discussion.

### Fix a typo or improve wording
Submit a pull request directly. No issue needed for small edits.

### Add a new section or substantially expand existing content
Open an issue first to discuss the addition before writing. This avoids duplicated effort and keeps the guide focused.

### Share a resource or reference link
Open an issue with the URL and a brief description of why it's relevant. We'll review and add it to the References section.

---

## Pull Request Guidelines

1. Fork the repo and create a branch: `git checkout -b my-contribution`
2. Make your changes to `docs/guide.md`
3. Keep the voice consistent — second person ("you"), direct, practical
4. Submit a pull request with a clear description of what you changed and why

---

## Content Guidelines

- **Be specific and actionable.** Vague advice ("network more") is less useful than concrete steps ("connect with every developer and PM you've worked with on LinkedIn").
- **Write from experience.** Tips grounded in real job search experience are more valuable than general career advice.
- **Keep the tone honest.** This guide doesn't sugarcoat the market. Maintain that directness.
- **Avoid vendor promotion.** Tool references are fine when genuinely useful (Claude, GitHub Pages, etc.) but the guide should not read as advertising.

---

## Credits

All contributors who submit accepted changes will be acknowledged in the Credits section of the guide. Please include your name (or preferred handle) in your pull request if you'd like to be credited.
